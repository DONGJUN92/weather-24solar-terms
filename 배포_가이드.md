# 🚀 GitHub + Vercel 배포 가이드 (비전문가용)

이 폴더의 웹앱(`prototype/`)을 인터넷에 공개하는 방법입니다. **10~15분**이면 됩니다.
이미 제가 `git` 커밋과 Vercel 설정(`vercel.json`)까지 만들어 두었으니, 아래 **복사-붙여넣기**만 따라 하시면 됩니다.

> 준비물: GitHub 계정(무료), Vercel 계정(무료). 둘 다 없으면 먼저 가입하세요.
> - GitHub 가입: https://github.com/signup
> - Vercel 가입: https://vercel.com/signup → **"Continue with GitHub"**(깃허브로 가입) 추천

---

## 1단계. GitHub에 빈 저장소(repository) 만들기

1. https://github.com/new 접속
2. **Repository name**: 예) `weather-24solar-terms` (영문/숫자/하이픈만)
3. **Public**(공개) 선택
4. 아래 체크박스는 **모두 해제** ⚠️ (Add a README / .gitignore / license 전부 끄기 — 안 그러면 충돌납니다)
5. 초록색 **Create repository** 클릭
6. 다음 화면에 나오는 주소를 복사해 둡니다. 예: `https://github.com/내아이디/weather-24solar-terms.git`

---

## 2단계. 내 컴퓨터에서 GitHub로 올리기 (push)

**Windows에서 폴더 안 터미널 열기**: 파일 탐색기로 이 폴더(`기상·기후 AI 해커톤`)에 들어가
주소창에 `cmd` 입력 후 Enter → 검은 창(명령 프롬프트)이 이 폴더에서 열립니다.

그 창에 아래를 **한 줄씩** 붙여넣고 Enter (`<주소>`만 1단계에서 복사한 것으로 바꾸세요):

```bat
git remote add origin <주소>
git branch -M main
git push -u origin main
```

- 처음 `push` 할 때 **브라우저 창이 떠서 GitHub 로그인**을 요청합니다 → 로그인/승인하면 끝.
- 끝나면 GitHub 저장소 페이지를 새로고침 → 파일들이 올라가 있으면 성공 ✅

> 이미 커밋은 되어 있습니다. (혹시 "nothing to commit"이 아니라 커밋이 필요하다고 나오면
> `git add -A` → `git commit -m "first"` 후 다시 push 하세요.)

---

## 3단계. Vercel로 배포하기 (클릭 몇 번)

1. https://vercel.com/new 접속 (GitHub로 로그인된 상태)
2. **Import Git Repository** 목록에서 방금 만든 저장소 옆 **Import** 클릭
   - 저장소가 안 보이면 **Adjust GitHub App Permissions / Configure** 에서 그 저장소 접근을 허용
3. 설정 화면이 나오면 **아무것도 바꾸지 말고** (제가 `vercel.json`을 넣어둬서 자동 인식됩니다)
   맨 아래 **Deploy** 클릭
4. 1~2분 기다리면 🎉 축하 화면 + **방문 주소**(예: `weather-24solar-terms.vercel.app`)가 나옵니다.
5. **Visit** 버튼 또는 그 주소를 누르면 앱이 인터넷에 떠 있습니다. 모바일에서도 그 주소로 열립니다.

---

## 4단계. 나중에 내용을 고치면? (자동 재배포)

앱을 수정한 뒤 2단계 폴더의 터미널에서:
```bat
git add -A
git commit -m "수정"
git push
```
→ Vercel이 **자동으로 새 버전을 다시 배포**합니다. (별도 작업 불필요)

---

## ❓ 문제 해결

- **화면이 비어 보여요 / 옛날 화면이에요** → 브라우저 강력 새로고침(`Ctrl+F5`). 데이터에 `?v=숫자`
  캐시 방지가 걸려 있어 보통은 자동 갱신됩니다.
- **Vercel에서 404가 떠요** → `vercel.json`이 저장소 최상단에 함께 올라갔는지 확인(2단계에서 같이 push됨).
  또는 Vercel 프로젝트 Settings → General → **Root Directory**를 `prototype`으로 바꿔도 됩니다.
- **`git`이 없다고 나와요** → https://git-scm.com/download/win 설치 후 cmd 창을 새로 여세요.
- **비밀키 안전** → 기상청/Copernicus/NASA 키는 코드·저장소에 들어있지 않습니다(확인 완료).
  데이터 수집 스크립트는 키를 환경변수로만 받으므로 공개 저장소에 올라가도 안전합니다.
